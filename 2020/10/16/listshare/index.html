Pointer&LinkedList&Stack

1.☞哪取哪，c语言利器——指针

 在介绍链表和栈之前，我们需要稍微了解一下指针的概念。

 ｃ语言中，每当我们定义一个变量时，内存中将会划分出一块由若干个存储单元组成的区域，用于保存该变量的数据。而内存里每个存储单元都有各自的编号，称为“地址”。

 如果把内存看成一条街，每个存储单元就像一个房子。而地址就是房子的门牌号。

 指针也是变量，只不过指针存储的值比较特殊，它存储的是地址。也就是那些房子的门牌号。

  • 地址

指针使用

 下面我们简单介绍一下指针的使用。

 通过如下方式声明一个指向int类型变量的，名字叫做 ‘ptr’ 的指针变量:

int* ptr;

int a = 5;

ptr = &a;

// 当然最好直接初始化
int* ptr = &a;

 既然指针能指向int类型的变量，是不是也能指向别的？答案是肯定的。

 指针指向的类型可以是int, char, float, double, void等。也可以是自定义的结构体、函数和数组。

typedef struct listNode {
    int Val;
    struct listNode* Next;
} listNode;

typedef struct Stack {
    int* base;
    int* top;
    int stack_size;
} Stack;

// ptr表示的就是一个指向类型为listNode型结构体的指针
listNode* ptr = (listNode*)malloc(sizeof(listNode));

// p表示的是指向一个长度为５的int型顺序表的指针
int* p = (int*)malloc(sizeof(int)*5);

 我们可以通过如下方法访问指针指向的变量

// 输出　５
printf("%d",*ptr);

// 输出节点node的值val
printf("%d",*ptr->Val);

特别提醒 => 野指针

int* ptr;

// 没有给 ptr 初始化
// 此 ptr 为野指针
printf("%d",*ptr);

 如果没有给指针初始化，指针就会指向一个随机的地址。当我们访问这个地址时，就会出现一些很讨厌的错误。

 所以我们要这样做：

// 初始化若用不上
int* ptr　=　NULL;

// 需要用时，先赋值 再使用
ptr = &a;

// 用完记得指向 NULL
ptr=NULL;

使用malloc函数会自动初始化,但是有可能会分配失败。

2.扒扒链表的裤子

 下面会介绍一下链表，顺便把它和顺序表做一个对比。

 链表和顺序表都是线性表。线性表有两种存储结构，分别是链式存储和顺序存储。

 顺序存储叫顺序表。它的结构是一段连续的内存空间。

 链式存储的，称为链表。节点与节点之间通过指针相连。

 下面我们来看看链表的节点长啥样

// 一个简单单向链表的节点
typdef struct listNode{
    ElemType Val;
    struct listNode* Next;
} listNode;

// 创建表头
lsitNode* head = (listNode*)malloc(sizeof(listNode));

// 创建新节点
listNode* node = (listNode*)malloc(sizeof(listNode));

// 链接表头和新节点
head->Next = node;

 这是一个简单单向的链表节点的结构。节点与节点之间，通过节点成员next连接，连成一条线，就形成了链表。

 要学会链表最基本的使用，无非是建立链表（插入节点）、删除节点、查找节点。下面一一介绍。

链表的建立/增

 在实际应用中，我们不可能像上面展示的那样去一个个建节点。实在是太累了，而且做的是无用功。这个时候，for循环就要登场了！

// 通过迭代创建多节点（插在表尾）
// 要用一个工具人帮我们迭代
listNode* tmp = head;

for (int i=0;i<10;i++) {
    listNode* node=(listNode*)malloc(sizeof(listNode));
    if (node) return error; // 判断有 malloc 分配是否成功，malloc失败返回NULL。
    node->Val = i; // 初始化赋值
    tmp->Next = node; // 链接
    tmp = tmp->Next; // 工具人始终走到表尾准备连接。
}

 要是每次插入数据都要用一个工具人插在表尾，时间复杂度会随着链表节点增加而增加,这显然不是我们所希望的。（气抖冷，工具人什么时候才能站起来！）

 所以我们这里要用一另一种方式:插在表头

for (int i=0;i<10;i++) {
    listNode* node = (listNode*)malloc(sizeof(listNode));
    if (!node) return error;
    node->Val = i;
    node->Next = head->Next; // 这样是保留头节点在最开始，node->Next = head
    head->Next = node;       // head = node 这样是不保留头节点在最开始
}

 这样时间复杂度不仅从 O(n) 降为O(1)，同时也不用工具人了。（原来连工具人都做不了吗）

链表迭代\查找

 所谓的迭代\查找，说简单点就是要在链表中找一个节点。其实在上面已经有提到了，这里直接放代码。

// 需要一个工具人帮我们找
listNode* p = head;

// 使用 while 循环，能简单地走完整个链表
while(p!=NULL) p=p->next; // 只需在循环中加入条件判断，就能轻松找到节点了。

 对于单向链表，我们不得不使用工具人。因为迭代过程中指针要一直往后面的节点走。但是节点又没有指向前面节点的指针。如果头节点往后走了，前面的节点就找不到了。

 在一般使用中，就算是双向链表，也会设置工具人去迭代。因为我们总是希望马上找到第一个节点。

 这样的迭代操作，时间复杂度是O(n),会随着链表的节点的增加而增加。但是除了这样找，也没有其他办法。这是链表的一个特征。

链表删除

 有了查找的基础，下面来看看链表的删除。

// 下面演示删除特定值的节点
listNode* p = ptr;

if (ptr->Val == key){ // 首先要分析头节点是否为我们要删的节点
    ptr = ptr->Next;
    free(p);
}

return 0;

// 头结点不是要删的，则继续往后找，需要一个工具人迭代，另一个工具人辅助删除。
p = p->Next;
listNode* tmp = ptr;

while(p!=NULL && p->Val!=key){
    p=p->Next;
    tmp=tmp->Next;
    return 0;
}

if (p == NULL){
    return 0;
}

tmp->Next = p->Next;
free(p);

 注意这里的链表是单向链表。如果是双向的，删除可以不需要这么麻烦。可以通过node->Pre->Next=node->Next来一步到位。

 可以看到，链表删除一个节点只需要改变节点之间的指针变量，非常方便。整个删除流程（除了查找部分）的时间复杂度是O(1)。

链表与顺序表

 不妨先回顾一下顺序表的建立（插入）、查找、删除。

  • 插入:插在表尾时间复杂度为O(1),就是时不时长度不够了要realloc。

  • 查找:因为顺序表内存是连在一起的，而我们又知道一个节点所占用的内存块大小。所以可以很方便地使用 *(p+n) 直接访问。时间复杂度为O(1)。

  • 删除:顺序表删除需要移动节点，非常麻烦。时间复杂度为O(n)。

 一对比就很容易发现两者的优缺点了。链表插入和删除是强项，而顺序表的查找是强项。

单向链表的扩展

  • 双向链表

 既然有指向后面节点的指针，那当然也可以有指向前面节点的。

typdef struct listNode{
    int Val
    *lstNode Pre;
    *listNode Next;
} listNode;

  • 环形链表

 我们之前的链表，最后一个节点指向的都是NULL。其实也可以指向表头。

3.揭开栈的神秘面纱

 说到栈，容易让人联想到下面几组词：“后进先出”、“数据结构”、“全栈工程师”、“爆栈”、“Stack overflow”。似乎提到栈的都是一些很强的东西

栈结构

 从数据结构看，栈也是线性表，而且是操作受限的线性表。

 我们一般将栈的表尾称为栈顶，表头称为栈底。栈的插入和删除，规定只能在栈顶，也就是表尾进行。因此，栈就有了后进先出的特点。

 和线性表类似，栈也有两种存储方式：顺序存储和链式存储。这里就看看顺序栈吧，链式栈太容易实现就不看了。

typedef struct Stack {
    SElemType *base; // base 指向栈底，不动
    SElemType *top;  // top指向栈顶，移动
    int stacksize;   // 当前已分配的存储空间，以元素为单位，方便判断爆栈
} Stack;

// 建立栈
Stack stack = (Stack*)malloc(sizeof(Stack));

stack->base = (SElemType *)malloc(STACK_INIT_SIZE * sizeof(SElemType));

// 这里可以加上判断 base 是否为空的条件

stack->top = stack->base

stack->stacksize = STACK_INIT_SIZE;

 就结构上来说，顺序栈只比顺序表多了一个指向表尾的指针。

栈使用

 栈的插入节点叫做”push”，删除节点叫做”pop”。实际上和顺序表的插入删除大同小异，只是多了一个对栈顶指针的操作。代码如下：

// push 压栈过程
// 如果容量不够，需要 realloc
if (stack->top-stack->base == stack->stacksize){
    int* tmp = (int*)realloc(stack->base,(stack->stacksize+INCREMENT_SIZE)*sizeof(int));
    if (!tmp) return error;

    stack->base = tmp;
    stack->stacksize += INCREMENT_SIZE;
}

*(stack->top) = node; // 压入
stack->top++;         // 栈顶指针移向后一位

// pop 弹出
if (stack->top==stack->base){
    return error; // 栈为空
}

stack->top--;

tmp = *(stack->top);

 栈的结构和线性表大同小异，不过它的思想需要记住。因为后进先出的特性，天然地适合解决某些问题。

  • 有效括号

  • 全排列（深度优先遍历）

代码运行时的内存结构

 计算机运行一个程序时，会给这个程序分配一段可使用的内存,如图所示：

  • Text Segment:代码段，存储指令

  • Initialized Data Segment:(data segment) 数据段，初始化的存储全局变量和静态变量

  • Uninitialized Data Segment:(block started by symbol) bss段，存储未初始化的或初始化为0的全局变量和静态变量。

  • Stack:这就是我们说的栈，这个区存储局部变量(包括main函数里的)，还和函数调用相关

  • heap：堆。用户可以通过malloc和free来操作这个区域。

 你是否会好奇，为何前文的所有示例代码，都用了malloc动态内存分配？把下面的代码复制到本地运行，你就懂了。

#include<stdio.h>
#include<stdlib.h>

typedef struct listNode {
    int Val;
    struct listNode* Next;
} listNode;

void hello(listNode** ptr){
    listNode node={100,NULL};
    (*ptr)->Next = &node;
}

void hello2(){
    listNode node={200,NULL};
}

int main(){
    listNode* ptr = (listNode*)malloc(sizeof(listNode));
    hello(&ptr);
    hello2();
    printf("%d",ptr->Next->Val);
    return 0;
}
